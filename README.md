# Python-Compiler

Учебный компилятор Python-подобного языка на C++20, собранный вокруг `Flex`, `Bison`, собственного AST, семантического анализа и генерации JVM bytecode.

Курсовой проект по дисциплине «Теория языков программирования и методы трансляции», посвящённый разработке компилятора для подмножества Python.

## Обзор

Проект принимает исходный код на подмножестве Python, разбирает его в синтаксическое дерево, проводит семантическую проверку и генерирует `.class`-файлы, которые затем можно посмотреть через `javap` и выполнить через `java`.

Пайплайн проекта выглядит так:

```text
Исходный код
  -> Лексический анализ (python.l)
  -> Синтаксический анализ (python.y)
  -> AST / дерево разбора
  -> Семантический анализ
  -> Генерация JVM bytecode
  -> Выполнение __PROGRAM__.class
```

## Ключевые возможности

- Лексический анализ на `Flex` с поддержкой `INDENT` / `DEDENT`, строк, литералов, комментариев и Python-подобной чувствительности к отступам.
- Синтаксический анализ на `Bison` с построением собственного дерева разбора.
- Семантическая обработка AST: таблицы классов, методов, полей, проверка идентификаторов и сигнатур вызовов.
- Генерация JVM bytecode для пользовательской программы и запуск результата через Java.
- Поддержка визуализации дерева в формате `DOT` через `Graphviz`.

## Поддерживаемые конструкции языка

По грамматике и тестам в репозитории проект покрывает следующий набор конструкций:

- переменные и присваивания;
- арифметические, логические и сравнительные выражения;
- `if / elif / else`;
- `while` и `for`;
- `try / except / else / finally`;
- функции, `return`, позиционные и именованные аргументы;
- классы и наследование;
- вызовы методов и обращения к атрибутам;
- списки, индексацию, slicing и list comprehensions;
- `lambda`-выражения;
- встроенные вызовы вроде `print`, `input`, `range`, а также часть строковых методов.

Это именно учебный компилятор подмножества Python, а не реализация полного языка Python.

## Рамки курсового проекта

По карточке курсовой работы и пояснительной записке проект нацелен на реализацию следующих возможностей языка:

- локальные переменные встроенных типов;
- числовые, строковые и символьные литералы;
- арифметические операции, сравнения и присваивания;
- ветвления и циклы;
- функции и `return`;
- ввод и вывод через консоль;
- списки как аналог одномерных и многомерных массивов;
- логические операции;
- операции над строками.

Именно этот учебный scope хорошо совпадает с тем, что реально видно в `python.l`, `python.y`, тестах и семантическом анализаторе проекта.

## Требования

Для сборки и запуска в текущем виде нужна Windows-среда:

- Visual Studio 2022 с toolset `v143`;
- установленный JDK, чтобы были доступны команды `javac`, `javap` и `java`;
- `Flex/Bison` и `Graphviz` уже лежат в репозитории в виде локальных зависимостей.

## Сборка

Самый прямой способ:

1. Открыть [Python-Compiler.sln](./Python-Compiler.sln) в Visual Studio.
2. Выбрать конфигурацию `x64 Debug` или `x64 Release`.
3. Собрать проект `Python-Compiler`.

Проект уже настроен на работу с `win_flex_bison`, поэтому `python.l` и `python.y` используются как источники для генерации лексера и парсера внутри сборки.

## Запуск

### Режим Debug

В `Debug`-режиме программа читает входной файл из [Python-Compiler/input.txt](./Python-Compiler/input.txt), поэтому достаточно:

1. Изменить содержимое `input.txt`.
2. Запустить проект из Visual Studio.

### Режим Release / CLI

В `Release`-сборке `main.cpp` ожидает два аргумента:

```bash
Python-Compiler.exe <input-file> <output-name>
```

Первый аргумент используется как путь к входному файлу. Второй аргумент зарезервирован под имя выходного `.dot`-файла для дерева, хотя соответствующий блок генерации сейчас закомментирован в коде.

## Результат работы

Во время выполнения проект:

- печатает сообщения лексера и парсера;
- запускает семантическую обработку;
- компилирует `__BASE__.java`;
- генерирует `__PROGRAM__.class`;
- показывает байткод через `javap -verbose`;
- запускает итоговую программу через `java`.

## Семантические проверки

По коду и сопровождающим материалам проект не ограничивается парсингом и ловит типичные семантические ошибки:

- вызов неизвестной функции или метода;
- использование неопределённой переменной;
- неверное количество аргументов при вызове;
- попытка вернуть неопределённое значение;
- некорректное использование выражений в `for`;
- ошибки доступа к полям и методам объектов.

Такие проверки напрямую видны в [Python-Compiler/semantic/fill_tables.cpp](./Python-Compiler/semantic/fill_tables.cpp), где выбрасываются подробные `runtime_error` с диагностикой.

## Примечания

- Проект ориентирован на демонстрацию устройства компилятора, а не на совместимость со всем Python.
- В кодовой базе уже есть задел под генерацию `DOT`-графов и визуализацию через `Graphviz`.
- Генерация кода нацелена на JVM, что делает проект особенно интересным как hybrid pipeline: Python-like frontend + Java bytecode backend.
- Поддержка классов в проекте реализована не полностью: эта часть не была доведена до финальной сдаваемой версии и не должна восприниматься как завершённая возможность компилятора.
- Материалы из пояснительной записки и карточки курсовой хорошо подтверждают назначение репозитория: это именно учебный компилятор под заданный набор возможностей, а не general-purpose Python implementation.
